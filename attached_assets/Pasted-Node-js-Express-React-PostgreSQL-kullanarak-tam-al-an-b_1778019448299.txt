Node.js + Express + React + PostgreSQL kullanarak tam çalışan bir admin panel geliştir.

Projenin amacı:
Station Satcom bayi panelindeki Rated CDRs trafik/kullanım verilerini kendi admin panelimizde görüntülemek istiyoruz. Station Satcom API vermiyor. Bu yüzden sistem, bizim kendi admin panelimizden girdiğimiz Station Satcom kullanıcı adı ve şifresiyle https://stationsatcom.satcomhost.com/ adresine giriş yapacak, https://stationsatcom.satcomhost.com/ratedCdrs.aspx sayfasındaki Rated CDRs tablosunu okuyacak ve verileri PostgreSQL veritabanına kaydedecek.

ÖNEMLİ:
Station Satcom kullanıcı adı ve şifresi .env dosyasında tutulmayacak.
Station Satcom bilgileri admin panelden girilecek ve PostgreSQL’de güvenli şekilde saklanacak.
.env içinde sadece teknik sistem değişkenleri olabilir:
- DATABASE_URL
- JWT_SECRET
- ENCRYPTION_KEY
- APP_PORT
- NODE_ENV

Station Satcom portal bilgileri kesinlikle .env içinde olmayacak.

Teknoloji:
- Backend: Node.js + Express
- Frontend: React
- Database: PostgreSQL
- ORM: Prisma tercih edilebilir
- Scraper/Otomasyon: Playwright Chromium
- Admin auth: JWT veya güvenli session
- Admin şifreleri bcrypt ile saklansın
- Station Satcom portal şifresi PostgreSQL’de düz metin tutulmasın, AES-256-GCM gibi güvenli bir yöntemle encrypt edilerek saklansın
- Encryption key .env içindeki ENCRYPTION_KEY ile yönetilsin

KIT numarası / ICCID alanı kesinlikle çekilmeli.

Admin panelden Station Satcom hesap bilgilerini kaydedelim, “Bağlantıyı Test Et” diyelim, sonra “Şimdi Senkronize Et” diyerek Rated CDRs tablosundaki KIT numaraları, kullanım GB bilgisi, dönem, fiyat ve CDR ID gibi verileri PostgreSQL’e kaydedelim ve panelde görelim.

4. portal_url adresine git.

9. Doğrudan /ratedCdrs.aspx sayfasına git.

Öncelikli grid selector:
#ctl00_ContentPlaceHolder1_gvRatedCdr

Fallback selectorlar:
- [id*="gvRatedCdr"]
- [id*="RatedCdr"]
- table[class*="dxgv"]
- body içinde KITP ile başlayan linkler

Veri çekme yöntemi:
- Önce grid içinde tüm a linklerini tara
- Text değeri KITP ile başlıyorsa bunu kit_no olarak kabul et
- Her KITP linkinin en yakın tr satırını bul
- O tr içindeki tüm td hücrelerini sırayla oku
- Boşlukları normalize et
- raw_row_data olarak tüm hücreleri JSONB kaydet
- Sonra mümkün olduğunca kolon mapping yap

Örnek Playwright extraction yaklaşımı:

const rows = await page.evaluate(() => {
  const grid =
    document.querySelector('#ctl00_ContentPlaceHolder1_gvRatedCdr') ||
    document.querySelector('[id*="gvRatedCdr"]') ||
    document.querySelector('[id*="RatedCdr"]') ||
    document.body;

  const links = Array.from(grid.querySelectorAll('a'));
  const kitLinks = links.filter(a => {
    const text = (a.textContent || '').trim();
    return text.startsWith('KITP');
  });

  return kitLinks.map(a => {
    const tr = a.closest('tr');
    const cells = tr
      ? Array.from(tr.querySelectorAll('td')).map(td =>
          (td.textContent || '').replace(/\s+/g, ' ').trim()
        )
      : [];

    return {
      kit_no: (a.textContent || '').trim(),
      raw_row_data: cells
    };
  });
});

Kolon mapping:
İlk aşamada indexler tam oturmayabilir. Bu yüzden raw_row_data mutlaka kaydedilsin.
Mapping hatalı olsa bile KIT numarası ve raw satır kaybolmasın.
Mapping için ayrı bir normalize fonksiyonu yaz.
